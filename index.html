Canon Protocol

A semantic data modeling system for defining versioned ontologies using YAML.

What is Canon Protocol?

Canon Protocol lets you define typed schemas — classes, properties, and instances — in simple .can.yml files. It draws on RDF and OWL concepts but is designed for developer-friendly authoring, versioned package management, and decentralized publishing.

  • Define data models with classes, properties, and type relationships
  • Import and compose packages from any publisher domain
  • Validate ontologies with a built-in rule engine
  • Publish packages to your own domain — no central registry required

Get Started with Agent Skills

The fastest way to start using Canon Protocol is with the canon Agent Skill. It teaches your coding agent to install the CLI, read Canon documentation, and author ontologies — all from a single skill file.

Add the skill to your project and your agent will know how to work with Canon Protocol:

Claude Code

Install the skill from the canon-protocol/skills repository:

claude mcp add-skill canon https://github.com/canon-protocol/skills/tree/main/skills/canon

Or manually place it in your project:

mkdir -p .claude/skills/canon
curl -o .claude/skills/canon/SKILL.md \
  https://raw.githubusercontent.com/canon-protocol/skills/main/skills/canon/SKILL.md

Gemini CLI

Add the skill to the shared agents directory:

mkdir -p .agents/skills/canon
curl -o .agents/skills/canon/SKILL.md \
  https://raw.githubusercontent.com/canon-protocol/skills/main/skills/canon/SKILL.md

A Minimal Example

A Canon package defines classes and properties in a single YAML file:

my-schema:
  type: Package
  publisher: mycompany.com
  version: 1.0.0
  imports:
    - publisher: canon-protocol.org
      packages:
        - package: core-rdf
          match: ^
          version: 1.0.1
          alias: rdfs
        - package: core-owl
          match: ^
          version: 2.0.2
          alias: owl
        - package: core-xsd
          match: ^
          version: 1.0.1
          alias: xsd

Person:
  type: rdfs.Class
  label: Person
  comment: An individual human being

fullName:
  type: owl.DatatypeProperty
  domain: Person
  range: xsd.string
