Canon Protocol

An open protocol for defining, versioning, and sharing semantic ontologies across distributed publishers.

View on GitHub

Why Canon Protocol?

Knowledge lives in the heads of domain experts, but there is no standard way to turn that knowledge into durable, machine-readable definitions that others can build on. Canon Protocol changes that.

Stable resource identifiers

Every class, property, and instance gets a persistent, globally unique identifier tied to a publisher domain. These identifiers don't break when systems change — they give people and machines a consistent way to refer to the same concepts.

Distributed publishing

Anyone with a domain can publish ontologies. There is no central authority deciding what types are valid. Domain experts — biologists, engineers, legal scholars — can develop their knowledge into formal schemas and share them directly from their own infrastructure.

Versioned ontologies

Ontologies evolve. Canon Protocol uses semantic versioning so publishers can improve their schemas without breaking consumers on earlier versions. Import what you need, pin to a version range, and upgrade on your own schedule.

Built for AI agents

When AI agents work with Canon ontologies, they are grounded in a shared vocabulary with formally defined types and relationships. The Canon CLI validates that data conforms to its schema — giving you guarantees about the shape of information rather than hoping the model got it right. Structured, validated data reduces ambiguity and gives agents less room to hallucinate.

Get Started with Agent Skills

The fastest way to start using Canon Protocol is with the canon Agent Skill. It teaches your coding agent to install the CLI, read Canon documentation, and author ontologies — all from a single skill file.

Add the skill to your project and your agent will know how to work with Canon Protocol:

Install the skill from the canon-protocol/skills repository:

claude mcp add-skill canon https://github.com/canon-protocol/skills/tree/main/skills/canon

Or manually place it in your project:

mkdir -p .claude/skills/canon
curl -o .claude/skills/canon/SKILL.md \
  https://raw.githubusercontent.com/canon-protocol/skills/main/skills/canon/SKILL.md

Any agent that supports the Agent Skills convention can use the shared .agents/skills/ directory:

mkdir -p .agents/skills/canon
curl -o .agents/skills/canon/SKILL.md \
  https://raw.githubusercontent.com/canon-protocol/skills/main/skills/canon/SKILL.md

Once installed, ask your agent to "create a new Canon ontology" or "explain Canon Protocol" and it will activate the skill automatically.

Example

Define an ontology

A Canon package defines classes and properties. This one describes people with addresses:

contacts:
  type: Package
  publisher: mycompany.com
  version: 1.0.0
  imports:
    - publisher: canon-protocol.org
      packages:
        - package: core-rdf
          match: ^
          version: 1.0.1
          alias: rdfs
        - package: core-owl
          match: ^
          version: 2.0.2
          alias: owl
        - package: core-xsd
          match: ^
          version: 1.0.1
          alias: xsd

Person:
  type: rdfs.Class
  label: Person
  comment: A person in the contacts directory

Address:
  type: rdfs.Class
  label: Address

fullName:
  type: owl.DatatypeProperty
  domain: Person
  range: xsd.string

email:
  type: owl.DatatypeProperty
  domain: Person
  range: xsd.string

hasAddress:
  type: owl.ObjectProperty
  domain: Person
  range: Address

street:
  type: owl.DatatypeProperty
  domain: Address
  range: xsd.string

city:
  type: owl.DatatypeProperty
  domain: Address
  range: xsd.string

Create instances

A separate package imports the ontology and creates typed instances with embedded resources:

company-directory:
  type: Package
  publisher: mycompany.com
  version: 1.0.0
  imports:
    - publisher: mycompany.com
      packages:
        - package: contacts
          match: ^
          version: 1.0.0

alice-smith:
  type: Person
  fullName: Alice Smith
  email: alice@mycompany.com
  hasAddress:
    street: 742 Evergreen Terrace
    city: Springfield

bob-jones:
  type: Person
  fullName: Bob Jones
  email: bob@mycompany.com
  hasAddress:
    street: 221B Baker Street
    city: London

Canon CLI

Install the CLI to validate, install packages, and inspect dependencies:

npm config set @canon-protocol:registry https://canon-protocol.org/npm
npm install -g @canon-protocol/cli

Install packages

# Install a package and its dependencies
canon install canon-protocol.org/core-rdf

# Install the full documentation bundle
canon install canon-protocol.org/docs-complete

Validate

# Validate a file — resolves imports and checks types
canon validate contacts.can.yml

Inspect dependencies

# Show the resolved dependency tree
canon deps contacts.can.yml

About

The name comes from "canonical" — the idea that something only needs to be defined once to become the globally referenceable, authoritative form of the thing it describes.

Canon Protocol was created by Paul Fryer in 2024 while building Genval, a startup focused on letting AI generate code from structured documentation. When existing schema formats like JSON Schema and RDF fell short on distributed publishing, versioning, and readability — he built Canon Protocol from scratch.
